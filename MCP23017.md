# MCP23017 I2C GPIO Expander — Heltec LoRa 32 V3 Integration Guide

## Overview

The MCP23017 is a 16-channel I2C GPIO expander. It shares the Heltec V3's existing I2C bus (already used by the OLED display), expanding available GPIO from ~10 safe pins to an arbitrary number via daisy-chaining up to 8 expanders (128 additional pins).

---

## 1. Wiring Configuration

The Heltec V3 (ESP32-S3) I2C bus is already wired to the OLED at `0x3C`. The MCP23017 joins this same shared bus.

| Component          | MCP23017 Pin | Heltec V3 GPIO | Note                        |
|--------------------|-------------|----------------|-----------------------------|
| I2C SDA            | Pin 13      | GPIO 17        | Shared with OLED display    |
| I2C SCL            | Pin 12      | GPIO 18        | Shared with OLED display    |
| Power (3.3V)       | Pin 9       | 3.3V           |                             |
| Ground             | Pin 10      | GND            |                             |
| Reset              | Pin 18      | 3.3V           | Pull-up (never drive LOW)   |
| Address A0/A1/A2   | Pins 15-17  | GND or 3.3V    | Sets I2C address 0x20–0x27  |

> **⚠️ Address Conflict Check:** The OLED uses `0x3C`. MCP23017 default with all address pins LOW is `0x20`. No conflict.

## 3. Integration with Existing Firmware

> **Critical:** Do NOT call `Wire.begin()` again in the MCP23017 init — the Heltec SDK already initializes the I2C bus for the OLED during `Heltec.begin()`. Calling it again will glitch the display.

```cpp
#include <Adafruit_MCP23X17.h>

Adafruit_MCP23X17 mcp;

// Call from DataManager or ScheduleManager init AFTER Heltec.begin()
void initMCP23017() {
  // Wire is already initialized by Heltec SDK on GPIO 17 (SDA) / 18 (SCL)
  if (!mcp.begin_I2C(0x20)) {
    Serial.println("MCP23017: NOT FOUND at 0x20");
    return;
  }
  Serial.println("MCP23017: Initialized OK");
  
  // Example: set GPA0-GPA7 as outputs (relay bank)
  for (int i = 0; i < 8; i++) {
    mcp.pinMode(i, OUTPUT);
    mcp.digitalWrite(i, LOW);
  }
  // Example: set GPB0-GPB7 as inputs with pullups (button/sensor bank)
  for (int i = 8; i < 16; i++) {
    mcp.pinMode(i, INPUT_PULLUP);
  }
}
```

## 4. Multi-Expander (Swarm GPIO) Configuration

Up to 8 MCP23017 chips can share the same bus by setting address pins differently:

| Address | A2 | A1 | A0 |
|---------|----|----|----|
| `0x20`  | L  | L  | L  |
| `0x21`  | L  | L  | H  |
| `0x22`  | L  | H  | L  |
| ...     | ...| ...| ...|
| `0x27`  | H  | H  | H  |

```cpp
Adafruit_MCP23X17 mcp0, mcp1; // Two expanders = 32 extra GPIO
mcp0.begin_I2C(0x20);
mcp1.begin_I2C(0x21);
```

## 5. Roadmap Integration

This hardware capability maps directly into the **LoRaLink Suite** feature roadmap:

| Feature | How MCP23017 Enables It |
|---|---|
| **Heterogeneous Device Support** | Devices with MCP23017 expand GPIO capacity without new hardware |
| **APC (Admin Pin Control)** | APC pins can now reference `mcp:0:3` style addresses, not just local GPIO |
| **Dynamic Task Scheduler** | `SCHED ADD` commands can target MCP pins via extended pin naming scheme |
| **Swarm Deploy** | Each node can carry a different number of MCP expanders; registry tracks capability |
| **Interrupt-Driven Events** | MCP23017 has an `INTA`/`INTB` output that can be wired to a native ESP32 GPIO for ISR-driven port reads |

### Extended Pin Naming Scheme (proposed)

```
PIN_MCP_0_4  →  MCP expander #0 (0x20), GPA4
PIN_MCP_1_12 →  MCP expander #1 (0x21), GPB4
```

This integrates natively with the existing `getPinFromName()` / `GetPinName()` abstraction in `CommandManager`/`DataManager`, just requiring a prefix check.

## 6. Interrupt Wiring (for Event-Driven Architecture)

Wire `INTA` → any free ESP32 GPIO (e.g. GPIO 38) for ISR-driven change detection:

```cpp
// In ScheduleManager::init() after MCP init:
pinMode(38, INPUT_PULLUP);
attachInterrupt(38, mcpInterruptHandler, FALLING);

// ISR (IRAM_ATTR):
void IRAM_ATTR mcpInterruptHandler() {
  // Set flag; process in main task
  _mcpInterruptFlag = true;
}
```

This aligns directly with the **ISR enhancement plan** already underway.
